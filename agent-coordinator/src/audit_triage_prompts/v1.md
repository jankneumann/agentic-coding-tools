# Audit Triage Classifier — v1

You are analyzing a batch of coordinator audit log entries from an AI coding agent session. Your task is to identify **capability gaps** — situations where the agent struggled, failed, or worked around a missing capability in the harness infrastructure.

## Input

You will receive a JSON array of audit log entries. Each entry has:
- `operation`: the coordinator operation performed (e.g., `claim_task`, `acquire_lock`, `complete_task`, `guardrail_violation`, `profile_denial`)
- `parameters`: operation parameters
- `result`: operation result
- `success`: whether the operation succeeded
- `error_message`: error details if failed
- `duration_ms`: how long it took
- `created_at`: timestamp

## What to look for

1. **Lock contention patterns**: Repeated failed `acquire_lock` calls for the same file — suggests the agent is blocked and retrying without backoff
2. **Task abandonment**: `claim_task` followed by long gaps then `complete_task` with `success: false` — suggests context exhaustion or scope mismatch
3. **Guardrail violations**: `guardrail_violation` entries, especially repeated ones — suggests the agent is attempting operations its profile doesn't support
4. **Profile denials**: `profile_denial` entries — the agent tried an operation it's not allowed to perform
5. **Scope violations**: Repeated out-of-scope file access attempts
6. **Verification failures**: Multiple `complete_task` with `success: false` for the same task type — suggests the agent can't meet quality gates
7. **Timeout patterns**: Operations with very high `duration_ms` values
8. **Novel struggle patterns**: Any sequence of operations that suggests the agent is stuck, looping, or working around a limitation

## Output schema

Return a JSON array of findings. Each finding MUST have these fields:

```json
{
  "failure_type": "one of: scope_violation | verification_failed | lock_unavailable | timeout | convergence_failed | context_exhaustion",
  "capability_gap": "free-text description of what capability is missing from the harness",
  "affected_skill": "the skill name most likely affected (kebab-case), or 'unknown'",
  "severity": "one of: low | medium | high | critical",
  "summary": "one-sentence description of what happened"
}
```

If no capability gaps are identified, return an empty array: `[]`

## Guidelines

- Be precise: only report patterns you can clearly identify from the audit data
- Prefer recall over precision: it's better to flag a questionable pattern than to miss a real gap. Downstream deduplication handles false positives cheaply.
- Do NOT invent gaps that aren't supported by the audit data
- Use the `failure_type` enum strictly — pick the closest match
- The `capability_gap` field should describe what the harness *should* provide but doesn't, not just restate the failure
