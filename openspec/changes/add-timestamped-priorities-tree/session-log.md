- Codeviz path convention <YYYY-MM-DD>/<run-id>.json (date folder + run-id filename): rejected because Optimized for single-file events; doesn't fit multi-file output without nesting or filename suffixes. The openspec archive idiom (per-run directory) fits multi-file output naturally.
- Symlink for latest.{md,json}: rejected because Posix-only; surprises archive consumers, GitHub web-UI downloads, and CI symlink-handling. Hybrid (symlink + fallback file) would create two ways to be wrong.
- Decision A1: Delete past-retention runs: rejected because Operator chose A2 (archive) at Gate 1. Deletion would be irrecoverable; the audit-trail value of a few MB/year is worth the bloat.
- Decision B1: Delete stale openspec/changes/prioritized-proposals.md: rejected because Operator chose B2 (legacy entry) at Gate 1. Preserves the May 2026 historical state in the new tree.

### Trade-offs
- Accepted ~5 KB duplication per run (dated artifact + latest flat file) over Symlink single-source-of-truth elegance because Cross-platform compatibility is worth the negligible duplication cost.
- Accepted ~2 MB/year sustained repo growth from archive accumulation over Zero-bloat deletion because Audit trail value is worth the bloat; text artifacts compress and de-dupe well in git.
- Accepted Inline header construction now, migration debt later over Waiting for skills/shared/artifact_header.py because Roadmap-planned helper does not exist; blocking on it would stall a small win. On-disk schema is identical to the future helper's, so migration is trivial.
