### Diagnostic Patterns (new this session)

- **GitHub Actions infra outage as a CI-failure class**: `setup-uv` action 404s on `codeload.github.com` clustered at the same minute across 13 failed jobs on #188. Pattern signature: same failure step, same minute, multiple unrelated PRs. Diagnostic: check Action source-fetch step, not the workflow logic. Resolution: wait for outage, re-run. Fourth failure class alongside transient/PR-specific/stale-merge.
- **gitleaks scans the full PR commit range**: Renaming a flagged string at HEAD doesn't clear the finding — gitleaks attributes it to the introducing commit, which remains in the PR's diff range. Inline `# gitleaks:allow` is silently ignored on 8.24.x (see `feedback_gitleaks_allowlist_singular`). The reliable fix is adding the introducing commit SHA to `.gitleaks.toml`'s `commits = [...]` allowlist with a comment explaining what the string is. We had to discover this after two failed attempts (inline comment → rename at HEAD).
- **`status=IN_PROGRESS conclusion=SUCCESS` is GraphQL lag, not a stall**: GitHub's check status API can show a workflow as in-progress while its conclusion is already SUCCESS. The merge endpoint uses the *true* workflow conclusion, so `gh pr merge` succeeds even when the status API still reports in-progress. Pattern: if conclusion is set, merge anyway — don't wait for status reconciliation.
- **Build-context path invariants in static checkers**: `check_docker_imports.py` was extracting `src.split("/")[0]` (e.g. `agent-coordinator` from `agent-coordinator/foo/`) as the import name. The destination (`/app/<name>/`) is the invariant under build-context changes. Fix: parse `dest`, not `src`. Regression test: `test_collect_copies_with_prefixed_source_uses_destination`.
- **Dockerfile path-dep COPY destinations matter**: For `uv` workspaces with path deps (`directory+../packages/gen-eval`), the COPY destination must match the resolved path *from `/app`*. Putting it at `/workspace/packages/gen-eval` breaks resolution; `/packages/gen-eval` works because `/app/../packages/gen-eval = /packages/gen-eval`.
- **`AuditEntry` is a dataclass, not a dict**: New `coordination_api.py` event-aggregation code called `.get("details")` on what's actually a `@dataclass`. mypy caught it, ruff didn't. Use `.result`. (`AuditEntry.result: dict[str, Any] | None`)
