### Observations

- **`dependency-audit-coordinator` broken-on-main pattern**: 5 PRs (#202, #201, #200, #199, #195) showed identical `dependency-audit-coordinator` failures because main's pinned pyjwt 2.12.1 had 4 unpatched CVEs (PYSEC-2026-175/177/178/179). Diagnosed before merging — merged Dependabot's #182 (pyjwt ≥2.13.0) first, then `refresh-branch` on all 5 UNSTABLE PRs to trigger fresh CI against the fixed base. Classic stale-merge variant: `rerun-checks` would have re-failed identically; `refresh-branch` was the right escape hatch.
- **Mergeability race**: First merge attempt on any PR shortly after the previous merge frequently returns `can_merge: false` because GitHub hasn't recomputed mergeStateStatus yet. Retry-2-3 with a 2s sleep is reliable. Worth surfacing this as a `merge_pr.py` enhancement: built-in retry on the "mergeability not yet computed" pattern.
- **Codex bot threads pin to old commit SHAs**: PR #196 had 3 "unresolved" P2 threads whose target code had been fixed before the bot's threads were posted (or in subsequent commits). GitHub doesn't re-evaluate the threads against new commits — they stay "unresolved" until manually closed. The GraphQL `resolveReviewThread` mutation works fine for batch-closing.
- **Codex P1 vs P2 reliability**: This pass had 2 P1s (#191 YAML, #187 review-artifacts) and 4 P2s. All 6 findings were real bugs (not false positives), but P1s mapped to "code would fail at parse/runtime" while P2s mapped to "validator gap" / "docs out of sync" / "cross-tier behavior break". The bot's P1/P2 split was a useful triage signal.
- **Sequential dep-bump bottleneck**: `pip` Dependabot PRs touching the same `pyproject.toml` (#185/#184/#183) must be merged sequentially with `refresh-branch` between each — only one can be merged before the others become stale. Worth experimenting with Dependabot's `combined-dependencies` grouping for this directory.
- **"Already fixed in HEAD" pattern**: 3 of 6 P2 findings on PRs we triaged were already fixed in the PR's head commit — the bot just hadn't re-evaluated. Cheap fix: before writing a code fix, always grep the actual lines first. Often what looks like a TODO is a closed loop with a stale comment.
